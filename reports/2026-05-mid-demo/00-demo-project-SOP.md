# 「示範林班 2026」demo project 建構 SOP

> 工作坊前 admin 須先建好 demo project + 樣區 + 立木，學員才能在情境 1 直接看到「示範林班 2026」並進行情境 2-6 的操作練習。
> 預估執行時間：**30-45 分鐘**（含 AI 辨識等待時間）

---

## 預備檢查（5 分鐘）

- [ ] **重貼 API key**（最高優先）：
  1. 開 https://forestry-rs-monitor.web.app
  2. admin 登入 → 任一專案 → 任一樣區 → 任一立木 → 「+ 新立木」
  3. 點「📸 AI 辨識」→ 右上「⚙️ 編輯全域設定」
  4. 貼新的 PlantNet key + Anthropic key
  5. 「💾 儲存到全域」
  6. 驗證：用任一張葉照測試辨識能跑出 top-3
- [ ] 已準備學員 email 清單（建議 ≤ 30 人，超過用「公開讀」模式）
- [ ] 知道工作坊現場 GPS 位置（學員會在哪裡建他們自己的樣區）
- [ ] PlantNet API 配額確認 ≥ 100/day 餘額（500/day free，預留學員實作用量）

---

## Step 1 — 建專案（3 分鐘）

1. admin Google 登入 → 我的專案 → 點「+ 新專案」
2. 填欄位：

| 欄位 | 值 |
|---|---|
| 專案 code | `DEMO-2026-001` |
| 專案名稱 | `示範林班 2026` |
| 主持人 PI | （admin 自己 uid） |
| 啟用模組 | ✅ 樣區 / ✅ 立木 / ✅ regen / ✅ understory / ✅ 野生動物 / ⬜ harvest |
| 方法學 plotShape 預設 | rectangle |
| 方法學 plotAreaOptions | 0.05 ha (20×25), 0.1 ha (20×50) |
| 方法學 dimensionType | dual-slope（雙軸坡度）|
| 是否公開讀 | （視學員人數決定）|

3. 「Submit」
4. 確認專案出現在「我的專案」列表

---

## Step 2 — 加學員為成員（10 分鐘，視人數）

> ⚠ 學員必須先用該 email 登入過 PWA 至少一次（Firestore `users/{uid}` doc 才存在），admin 才加得到。建議工作坊報到時引導學員「先登入一次」。

1. 進「示範林班 2026」→ ⚙ 設定 → 成員管理
2. 「+ 加成員」→ 填學員 Google email → 設角色 = `surveyor`
3. 重複加完所有學員
4. 加 1-2 位協助講師為 `reviewer`
5. （備援）若有學員忘記事先登入：
   - 引導他們現場登入後告知 admin
   - admin 即時加進去
   - 學員 F5 重新整理

**替代方案**：人數 > 30 直接設「公開讀」（任何登入帳號都看得到此專案，但只有正式成員能寫資料）。Firestore rules 預設不支援 — 需臨時改 rules（不建議，預設方法為佳）。

---

## Step 3 — 建 demo 樣區（10 分鐘）

> 共 2 個樣區，示範矩形 + 圓形兩種形狀。

### Plot 1 — 矩形 20×25（典型紙漿廠 SOP）

1. 進專案 → 樣區分頁 → 「+ 新樣區」
2. 填：

| 欄位 | 值 | 備註 |
|---|---|---|
| 樣區編號 | `DEMO-2026-001-001` | 自動續號 |
| GPS | （現場抓 / 手動輸入）| 建議用工作坊會場真實座標，學員看得懂 |
| 形狀 | rectangle | |
| 寬 (m) | 20 | 沿坡 |
| 長 (m) | 25 | 沿等高線 |
| 寬邊坡度 (slopeWidthDeg) | `15°` | demo 用中度坡 |
| 長邊坡度 (slopeLengthDeg) | `2°` | 接近水平 |
| 設置日期 | 工作坊前 1-2 週 | |
| 樣區照片 | 上傳 1 張會場照 | |

3. 系統會即時算：areaSlope_m2 ≈ 500.0、areaHorizontal_m2 ≈ 482.9
4. Submit

### Plot 2 — 圓形 0.05 ha

1. 「+ 新樣區」
2. 填：

| 欄位 | 值 |
|---|---|
| 樣區編號 | `DEMO-2026-001-002` |
| GPS | （另一個位置，或同會場附近）|
| 形狀 | circle |
| 半徑 (m) | 12.62 (= sqrt(500/π)) |
| 平均坡度 | `8°` |

3. Submit

## Step 4 — 建 demo 立木（15 分鐘）

> 共 5-10 棵，刻意涵蓋 3 種公式徽章顏色 + 1 棵 AI 辨識，展示透明度與功能。

進 Plot 1 → 立木 subtab → 逐筆 `+ 新立木`：

| # | 樹種（中名）| DBH (cm) | H (m) | 局部 X | 局部 Y | 活力 | 徽章 | 備註 |
|---|---|---|---|---|---|---|---|---|
| 1 | 紅檜 | 35 | 18 | 5 | 5 | 健康 | 🟢 | species-specific（陳朝圳 1985）|
| 2 | 台灣杉 | 28 | 22 | 8 | 7 | 健康 | 🟢 | species-specific |
| 3 | 樟樹 | 42 | 16 | 3 | 12 | 健康 | 🟡 | tree-type fallback（闊葉樹）|
| 4 | 山黃麻 | 18 | 12 | 15 | 8 | 受損 | 🟡 | 註明病蟲害 |
| 5 | 大葉桉（外來）| 30 | 14 | 12 | 18 | 健康 | 🟠 | generic IPCC tier-1 |
| 6 | 楓香 | 25 | 15 | 7 | 20 | 健康 | 🟢 | （視字典實際情況可能 🟡）|
| 7 | 紅檜 | 50 | 20 | 18 | 22 | 健康 | 🟢 | 大徑木 |
| 8 | **AI 辨識用** | 視結果 | 視測 | 10 | 10 | 健康 | 視結果 | 用 AI 辨識任一片葉子，照片自動寫入 |

進 Plot 2 加 2-3 棵類似結構（縮小版）即可。

**展示重點**：
- 紅檜 vs 山黃麻 vs 大葉桉 → 公式徽章顏色不同 → demo 「公式來源透明度」價值
- 上傳特徵照（任意室內物品代拍 1-2 張即可）→ tree.photos 不為空
- AI 辨識用一片真葉照（手邊樹葉 / 印好的葉照），照片會自動入 photos

---

## Step 5 — 標 verified 讓 plot 進入 review 階段（5 分鐘）

> reviewer QAQC 流程需要 plot.qaStatus = 'verified' 才會自動切換 project status。

1. 進 Plot 1 → 樣區概覽分頁 → 滑到底部 `qaStatus` 區塊
2. 點「✓ 標記 verified」
3. Plot 2 同上
4. 系統偵測到所有 plots verified → 自動把 project status 切到 `review`
5. reviewer 帳號登入後即可看到「🔍 審查（QAQC）」分頁

**驗證**：用 reviewer 帳號登入 → 進「示範林班 2026」→ 確認 QAQC 分頁出現

---

## Step 6 — 預跑一次 QAQC（5 分鐘）

讓示範時 reviewer 端有「已抽樣 + 已重測」的資料可看：

1. reviewer 帳號 → QAQC 分頁 → 設 config：
   - plot 抽樣比例：50%（2 個 plot 抽 1）
   - tree per plot：3
   - DBH 閾值：±2 cm
   - height 閾值：±1 m
2. 「💾 儲存設定」→ 系統隨機抽 1 plot + 3 trees
3. 對抽中的 3 棵樹依序填重測值（**故意有 1 棵超過閾值，產生 🔴 紅 badge 供示範**）
4. 對紅色 flag 點「accept」（保留作為示範）
5. **不要**點「合格簽發」（保留簽發按鈕的展示機會給工作坊現場）

---

## Step 7 — 確認 AI 辨識可正常執行（2 分鐘）

> 工作坊一旦 AI 失敗會非常難看。最後總驗收。

1. 進任一立木 → 「📸 AI 辨識」
2. 用手邊葉照（或網上下載 1 張高解析葉照）測試
3. 確認 5-15 秒內回 top-3 + 信心 % 三色徽章
4. 點選任一筆 → 樹種 picker 自動填入 + 照片自動入 tree.photos
5. ✅ 通過 → demo 完備

---

## 收尾 checklist

- [ ] 「示範林班 2026」可在「我的專案」看到
- [ ] 至少 2 個樣區（rectangle + circle）
- [ ] 至少 5 棵立木，含 🟢 / 🟡 / 🟠 三色徽章
- [ ] 1 棵立木有 AI 辨識照片
- [ ] 1 個 plot 已標 verified
- [ ] project status = `review`
- [ ] reviewer 端 QAQC 已有 1 plot + 3 trees 抽樣 + 重測（含 1 紅 badge）
- [ ] AI 辨識實機驗證通過
- [ ] 全部學員已加為成員（或公開讀已開）
