# 森林監測 App 需求規格（v1 MVP）

> 對應 FMP 範本「肆、經營規劃方案 — 監測」與「貳、現況分析 — 森林資源」

---

## 壹、使用者角色

### 一、調查員（surveyor）
- 在野外用手機/平板填表
- 只能看到自己調查的樣區（資料隔離）
- 可離線收集，回連自動同步

### 二、計畫主持人（pi）
- 看全部樣區、所有調查者資料
- 編輯任何資料
- 匯出 Excel/CSV
- 邀請新調查員加入

### 三、審查委員（reviewer）
- 唯讀，看儀表板與原始資料
- 可加註評論（v2 功能）

## 貳、功能模組（v1 MVP）

### 一、登入與帳號
- (一) Firebase Auth：Email/Password + Google OAuth
- (二) 首次登入填顯示名、機構、角色（角色由主持人核定）
- (三) 主持人在 admin 介面指派角色

### 二、樣區管理
- (一) 列表：顯示「我的樣區」/「全部樣區」（依角色）
- (二) 新增樣區
   1. 樣區編號（自動建議：`<案件代碼>-<3 位序號>`）
   2. 林班-小班（從專案邊界帶入候選）
   3. GPS 自動抓取（使用者按「定位」鍵）→ 顯示 WGS84 + TWD97 TM2
   4. 林班界檢核（GPS 是否落在計畫範圍內，否則警示但仍可存）
   5. 樣區形狀（圓 / 方）+ 面積
   6. 設置日期、概況照片（4 張：N/E/S/W）
   7. 備註

### 三、立木調查（樣區內子表）
- (一) 列表：本樣區所有立木個體
- (二) 新增/編輯個體
   1. 個體編號（樣區內自動序號）
   2. 樹種：autocomplete（連動 species-conservation-lookup 自動帶入學名 + 保育等級警示）
   3. DBH（cm）：必填，整數或一位小數
   4. 樹高 H（m）：必填
   5. 枝下高（m）：選填
   6. 活力等級：健康 / 衰弱 / 枯立 / 倒伏
   7. 病蟲害症狀（多選）：葉斑、潰瘍、蟲孔、空洞、菌害、無
   8. 標記方式：噴漆 / 號牌 / 無
   9. 個體照片
   10. 備註
- (三) 自動計算（即時）：胸高斷面積、單木材積、單木碳蓄積（套用 carbon-volume-calculator 公式）

### 四、自然更新調查（樣區內子表）
- (一) 列表：本樣區所有更新觀察記錄
- (二) 新增/編輯記錄
   1. 樹種（同立木 autocomplete）
   2. 苗高分級：< 30 cm / 30–130 cm / > 130 cm
   3. 該級數量
   4. 競爭植被覆蓋度（0–100%）
   5. 備註

### 五、即時分析儀表板
- (一) 進度地圖：樣區點位 + 林班界 + 完成度色階
- (二) 林分結構：
   1. 直徑分布直方圖（依樹種或全體）
   2. 樹種重要值（IV）排行
   3. 樣區小計表（株數、平均 DBH、平均 H、總斷面積、總材積、總碳量）
- (三) 健康度：衰弱/枯立比率
- (四) 更新指數：每樣區每樹種苗木密度（株/m²）

### 六、匯出
- (一) Excel：樣區/立木/更新 三 sheet
- (二) CSV：每集合一檔
- (三) Shapefile（v2）：樣區點位

## 參、非功能需求

| 面向 | 要求 |
|------|------|
| 離線 | 野外無訊號常態 → IndexedDB 暫存，回連自動同步 |
| 效能 | 單樣區立木 100 株以下時，列表開啟 < 1 秒 |
| 容錯 | GPS 抓不到 → 允許手填座標但加旗標 |
| 隱私 | 樣區位置精度 ≤ 1 m；非調查員看不到精確 GPS（v2 模糊化） |
| 可移植 | 零 build，CDN-only，任何人 fork 後改 firebase-config 就能跑 |

## 肆、與其他 modules 的對接

| 對象 | 連動方式 |
|------|---------|
| `carbon-volume-calculator` | 立木調查的材積/碳量計算公式來源 |
| `species-conservation-lookup` | 樹種 autocomplete 與保育等級警示 |
| `fmp-spatial-analysis` | 林班界 GeoJSON 來源、樣區點位輸出 |
| `forest-growth-yield` | 匯出資料可直接餵給此模組做生長收穫分析 |
| `multispectral-3d-forest-segmentation` | 樣區地真資料來源（Ground Truth） |
